## Goal

기존 Todo 앱에서 사용자가 전체, 미완료, 완료 항목을 필터링하고, 새로고침 후에도 할 일 목록과 완료 상태를 유지할 수 있게 한다.

## Background

현재 Todo 앱은 새로고침하면 항목이 사라지고, 목록이 길어졌을 때 완료되지 않은 항목만 확인하기 어렵다.

## Scope

### Included

- Todo 상태 필터 UI
- 브라우저 localStorage 기반 Todo 저장
- 기존 Todo 추가, 완료 토글, 삭제 흐름 보존

### Excluded

- 로그인과 사용자별 동기화
- 서버 저장소
- 마감일, 태그, 우선순위 기능

## Acceptance Criteria

- AC-001: 사용자는 전체, 미완료, 완료 필터를 전환하고 조건에 맞는 Todo 항목을 볼 수 있다.
- AC-002: 새 Todo 항목과 완료 상태는 브라우저 새로고침 후에도 유지된다.
- AC-003: 저장 기능이 추가된 뒤에도 삭제와 완료 토글이 계속 동작한다.
- AC-004: 필터 결과가 비어 있을 때 사용자가 현재 상태를 이해할 수 있다.

## Constraints

- 클라이언트 저장소만 사용한다.
- 백엔드 의존성을 추가하지 않는다.
- 기존 단일 페이지 Todo 앱 구조를 유지한다.

## Assumptions

- 기존 앱은 Todo 추가, 완료 토글, 삭제를 이미 지원한다.
- 앱은 localStorage를 사용할 수 있는 최신 브라우저에서 실행된다.

## Risks

- 사용자가 브라우저 저장소를 지우거나 실행 환경에서 localStorage가 비활성화될 수 있다.
- 저장된 데이터가 손상되면 초기 렌더링이 깨질 수 있다.
