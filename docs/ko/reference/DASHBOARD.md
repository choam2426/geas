# Dashboard

## 개요

Geas Dashboard는 프로젝트의 미션 진행 상황, 태스크 상태, 기술 부채, 이벤트 타임라인 등을 시각적으로 모니터링하는 데스크톱 애플리케이션이다. Tauri v2 기반으로 구축되었으며, Rust 백엔드가 `.geas/` 디렉터리의 상태 파일을 읽어 React 프론트엔드에 데이터를 제공한다.

다수의 프로젝트를 동시에 등록하여 관리할 수 있고, 파일 시스템 변경을 감지하여 자동으로 화면을 갱신한다.

## 설치 및 실행

### 사전 요구 사항

| 항목 | 버전 |
|------|------|
| Node.js | 18 이상 |
| Rust | stable |
| Tauri CLI | 2.x |

### 개발 모드 실행

```bash
cd src/dashboard
npm install
npm run tauri dev
```

`tauri dev` 명령은 Vite 개발 서버(포트 5173)와 Tauri 네이티브 창을 동시에 시작한다.

### 프로덕션 빌드

```bash
cd src/dashboard
npm run tauri build
```

빌드 결과물은 플랫폼에 따라 다음 형식으로 생성된다.

| 플랫폼 | 패키지 형식 |
|--------|------------|
| Windows | MSI, NSIS |
| macOS | DMG |
| Linux | DEB, AppImage |

### 창 기본 설정

- 기본 크기: 1200 x 800
- 최소 크기: 800 x 600
- 크기 조절 가능, 전체 화면 비활성

## UI 구성

Dashboard는 좌측 사이드바와 우측 메인 영역으로 나뉜다. 사이드바에서 프로젝트를 선택하면 메인 영역에 해당 프로젝트의 정보가 표시된다.

### 사이드바 (Sidebar)

프로젝트 목록을 표시하는 좌측 패널이다.

- **프로젝트 목록**: 등록된 프로젝트를 나열하며, 선택된 프로젝트는 강조 표시와 좌측 강조선으로 구분한다.
- **상태 아이콘**: `.geas/` 디렉터리가 없는 프로젝트는 경고 아이콘, 읽기 오류가 발생한 프로젝트는 에러 아이콘을 표시한다.
- **페이즈 뱃지**: 각 프로젝트의 현재 미션 페이즈를 색상 뱃지로 표시한다.
- **접기/펼치기**: 사이드바를 아이콘 모드로 접을 수 있다. 접힌 상태에서는 프로젝트 이름의 첫 글자와 페이즈 색상 점으로 표시한다. 화면 너비가 1024px 미만이면 자동으로 접힌다.
- **프로젝트 추가**: 하단의 "Add Project" 버튼으로 새 프로젝트를 등록한다.
- **프로젝트 제거**: 각 프로젝트에 마우스를 올리면 나타나는 X 버튼으로 제거한다. 확인 대화 상자가 표시된다.
- **새로고침**: 상단의 새로고침 버튼으로 전체 프로젝트 목록을 갱신한다.
- **미션 기록**: 상단의 시계 아이콘으로 Mission History 화면으로 이동한다.

### 프로젝트 대시보드 (Project Dashboard)

프로젝트를 선택하면 표시되는 메인 화면이다.

**헤더 영역**

프로젝트 이름과 경로를 표시한다.

**바로가기 버튼**

- **Timeline**: 이벤트 타임라인 화면으로 이동
- **Memory**: 에이전트 메모리 브라우저로 이동
- **Rules**: 프로젝트 규칙 뷰어로 이동

**Health Alert 배너**

`.geas/` 디렉터리의 health-check.json에 트리거된 신호가 있으면 빨간색 배너로 표시한다. 각 신호의 이름, 상세 내용, 필수 대응 조치를 보여준다.

**Currently Working 카드**

현재 작업 중인 태스크 정보를 표시한다. 녹색 펄스 점으로 활성 상태를 나타낸다.

- 단일 태스크 작업: 태스크 ID, 에이전트 이름, 파이프라인 단계를 표시
- 병렬 태스크 작업: 병렬 배치에 포함된 태스크 목록을 표시하고, 완료된 태스크는 체크 표시

**활성 미션 카드**

현재 활성 미션의 이름, 페이즈 뱃지, 진행률 막대를 표시한다. View Tasks, View Debt, Detail 버튼을 제공한다.

**통계 카드**

| 항목 | 설명 |
|------|------|
| Total Tasks | 전체 미션의 태스크 합계 |
| Debt Items | 기술 부채 항목 수 |
| Last Activity | 마지막 활동 시각 (상대 시간 표시) |

**미션 목록**

등록된 모든 미션을 나열한다. 활성 미션은 좌측 강조선으로 구분하고, 전체 태스크가 완료된 비활성 미션은 투명도를 낮춰 표시한다. 각 미션을 클릭하면 칸반 보드로 이동한다.

### 칸반 보드 (Kanban Board)

태스크를 상태별 열로 분류하여 표시하는 화면이다.

![칸반 보드](../../images/kanvanboard.png)

**기본 열 (7개 주요 상태)**

| 열 | 색상 | 설명 |
|----|------|------|
| Drafted | 회색 | 초안 작성됨 |
| Ready | 파란색 | 구현 준비 완료 |
| Implementing | 녹색 | 구현 진행 중 |
| Reviewed | 노란색 | 검토 완료 |
| Integrated | 파란색 | 통합됨 |
| Verified | 녹색 | 검증됨 |
| Passed | 밝은 녹색 | 최종 통과 |

**보조 상태 (Auxiliary States)**

기본 열에 해당하지 않는 태스크는 하단의 보조 상태 영역에 표시한다.

| 상태 | 아이콘 | 설명 |
|------|--------|------|
| Blocked | 금지 | 차단됨 |
| Escalated | 경고 | 에스컬레이션됨 |
| Cancelled | X | 취소됨 |
| Paused | 일시정지 | 일시 중단 |

**태스크 카드 (Task Card)**

각 카드에는 태스크 제목, ID, 리스크 레벨 뱃지가 표시된다. 현재 작업 중인 태스크는 녹색 테두리와 펄스 효과로 강조되며, 에이전트 이름과 파이프라인 단계가 추가로 표시된다. 병렬 배치에서 완료된 태스크는 녹색 체크 배경으로 구분한다.

**태스크 상세 모달 (Task Detail Modal)**

태스크 카드를 클릭하면 상세 모달이 열린다. 다음 정보를 확인할 수 있다.

- **기본 정보**: 태스크 ID, 상태, 리스크 레벨, 태스크 종류, 작업자 유형
- **목표 (Goal)**: 태스크의 목적
- **수용 기준 (Acceptance Criteria)**: 통과 조건 목록
- **스코프 표면 (Scope Surfaces)**: 영향을 받는 범위
- **의존성 (Dependencies)**: 선행 태스크 목록
- **구현 계약 (Implementation Contract)**: 계획된 작업 항목, 엣지 케이스, 비목표, 데모 단계
- **자체 점검 (Self Check)**: 신뢰도, 요약, 알려진 위험, 미테스트 경로
- **게이트 결과 (Gate Result)**: 단계별(Tier 0/1/2) 결과, 루브릭 점수, 차단 항목
- **챌린저 리뷰 (Challenge Review)**: 차단 여부, 요약
- **판정 (Verdict)**: 최종 판정, 근거, 되돌리기 대상
- **클로저 (Closure)**: 변경 요약, 리뷰 목록, 미해결 위험, 부채 항목
- **회고 (Retrospective)**: 잘된 점, 문제점, 교훈, 규칙/메모리/부채 후보
- **에비던스 (Evidence)**: 역할별 에비던스 목록 (에이전트, 요약, 판정, 파일 변경 목록)

**부채 패널 (Debt Panel)**

칸반 보드 하단에 해당 미션의 기술 부채 요약을 표시한다. 심각도별 수량을 색상으로 구분하여 보여준다.

### 기술 부채 상세 (Debt Detail Panel)

기술 부채 항목을 필터링하고 상세 정보를 확인하는 화면이다.

**필터**

- **심각도 필터**: critical, high, normal, low 중 선택 (다중 선택 가능)
- **종류 표시**: output quality, verification gap, structural, risk, process, documentation, operations별 수량 표시
- **상태 필터**: all, open, resolved 중 선택

**부채 항목 카드**

각 카드에는 심각도 뱃지, 종류 뱃지, 상태 뱃지, 제목, 설명, 부채 ID, 관련 태스크 ID가 표시된다. 카드를 클릭하면 상세 모달이 열려 전체 정보를 확인할 수 있다.

### 미션 기록 (Mission History)

프로젝트의 모든 미션을 카드 형식으로 나열하는 화면이다.

각 미션 카드에는 미션 이름, ID, 페이즈 뱃지, 활성 여부, 태스크 진행률, 생성일이 표시된다. 카드를 클릭하면 해당 미션의 칸반 보드로 이동한다. 각 카드의 Detail 버튼으로 미션 상세, Timeline 버튼으로 타임라인을 열 수 있다.

### 미션 상세 (Mission Detail View)

미션의 전체 명세, 설계, 리뷰 정보를 통합 표시하는 화면이다.

| 섹션 | 내용 |
|------|------|
| Mission Spec | 미션 목적, 완료 조건, 스코프, 수용 기준, 제약 조건, 위험 사항, 가정, 모호점, 영향 표면 |
| Design Brief | 선택된 접근법, 비목표, 검증 전략, 대안, 아키텍처 결정, 위험, 보존 목록, 미해결 가정 |
| Vote Rounds | 투표 참여자, 각 투표 내용과 근거, 결과 |
| Phase Reviews | 미션 페이즈별 리뷰 결과, 충족/미충족 기준, 위험 사항 |
| Gap Assessment | 스코프 전달 현황 (완전/부분/미전달), 의도적 삭감, 예상외 추가, 후속 권장 사항 |

메타 정보로 도메인 프로파일, 모드, 대상 사용자, 소스 등을 표시한다.

### 이벤트 타임라인 (Timeline View)

미션 또는 프로젝트 단위의 이벤트 로그를 시간순으로 표시하는 화면이다.

![토스트 알림](../../images/toast.png)

- 각 이벤트는 유형 뱃지, 태스크 ID, 에이전트 뱃지, 메시지, 타임스탬프로 구성된다
- 에이전트별로 고유 아이콘과 색상이 지정된다 (software-engineer, design-authority, qa-engineer, challenger, product-authority, orchestration-authority)
- 이벤트 유형별 색상 구분: task_started(파란색), task_resolved(녹색), gate_result(노란색), phase_complete(보라색) 등
- 페이지당 50개 이벤트로 페이지네이션된다

### 에이전트 메모리 브라우저 (Memory Browser)

프로젝트의 에이전트 메모리 파일을 열람하는 화면이다.

좌측에 에이전트 목록, 우측에 선택한 에이전트의 메모리 내용을 표시한다. `.geas/memory/agents/` 디렉터리의 내용을 읽어온다.

### 프로젝트 규칙 뷰어 (Rules Viewer)

프로젝트의 `.geas/rules.md` 파일 내용을 마크다운 렌더링하여 표시하는 화면이다. GFM(GitHub Flavored Markdown) 문법을 지원한다.

## 실시간 갱신

Dashboard는 두 가지 경로로 화면을 자동 갱신한다.

1. **파일 시스템 감시**: Tauri 백엔드가 `.geas/` 디렉터리의 변경을 감지하면 `geas://project-changed` 이벤트를 프론트엔드로 전송한다. 사이드바의 프로젝트 요약, 대시보드의 미션 목록, 칸반 보드의 태스크, 타임라인의 이벤트 등 모든 화면이 이 이벤트에 반응하여 갱신된다.

2. **토스트 알림**: 백엔드가 이벤트를 분류하여 `geas://toast` 이벤트로 전송한다. 2초 이내의 동일한 알림은 중복 제거된다.

| 토스트 유형 | 표시 스타일 |
|------------|------------|
| task_started | 정보 (파란색) |
| task_completed | 성공 (녹색) |
| gate_pass | 성공 (녹색) |
| gate_fail | 에러 (빨간색) |
| phase_changed | 정보 (파란색) |
| mission_completed | 성공 (녹색) |

각 토스트는 제목, 메시지, 진행률 막대를 포함하며, 일정 시간 후 자동으로 사라지거나 수동으로 닫을 수 있다.

## 프로젝트 등록

### 프로젝트 추가

1. 사이드바 하단의 "Add Project" 버튼을 클릭한다.
2. 대화 상자에 프로젝트 경로를 입력한다.
3. "Add" 버튼을 클릭하면 백엔드가 경로를 검증하고 프로젝트를 등록한다.

등록된 프로젝트 경로는 앱 재시작 후에도 유지된다. `.geas/` 디렉터리가 없는 프로젝트도 등록은 가능하나, 대시보드에 경고 상태로 표시된다.

### 프로젝트 제거

사이드바에서 프로젝트 이름 위에 마우스를 올리고, 나타나는 X 버튼을 클릭한다. 확인 대화 상자에서 승인하면 목록에서 제거된다. 프로젝트 파일 자체는 삭제되지 않으며, 대시보드 등록만 해제된다.

## 내비게이션

Dashboard는 브라우저 스타일의 뒤로가기 내비게이션을 지원한다. 각 화면 전환은 내비게이션 히스토리에 기록되며, "Back" 버튼으로 이전 화면으로 돌아갈 수 있다. 히스토리가 없는 경우 대시보드 메인 화면으로 이동한다.

화면 전환 시 페이드 인 애니메이션이 적용된다.

## 스크린샷

### 프로젝트 대시보드

![대시보드 메인 화면](../../images/dashboard.png)

좌측 사이드바에 프로젝트 목록, 우측에 선택된 프로젝트의 대시보드가 표시된다. 활성 미션의 페이즈, 진행률, 통계, 미션 목록을 한 화면에서 확인할 수 있다.

![칸반 보드](../../images/kanvanboard.png)

### 토스트 알림

![토스트 알림](../../images/toast.png)
