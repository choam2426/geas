# Debate 모드 가이드

## 사용 시기

Debate는 **코드 없는 구조적 의사 결정**을 위한 모드입니다. 에이전트들이 다음 사항을 토론하고 결정해야 할 때 사용하세요:

- 아키텍처 선택
- 기술 선정
- 설계 방향
- 전략적 pivot
- 트레이드오프 분석
- 프로세스 결정

Debate는 **DecisionRecord**를 생성합니다 — 구현 없이 추론과 합의만 도출합니다.

## 작동 방식

**1. 질문 구성**
- 결정해야 할 사항을 설명합니다
- 2–3가지 명확한 옵션을 식별합니다
- 컨텍스트를 제공합니다 (기존 코드베이스, 제약 조건, 왜 중요한가)

**2. 토론자 생성**
시스템이 입장을 논할 에이전트를 생성합니다:
- **Forge** — 기술적 근거를 들어 하나의 옵션을 지지하여 논쟁합니다
- **Critic** — 반대 논거를 제시하고 가정에 도전합니다
- **Circuit** — 백엔드/확장성 관점에서 양쪽을 평가합니다
- **Palette** — UX/프론트엔드 관점에서 양쪽을 평가합니다

**3. 종합**
논거가 요약됩니다. 트레이드오프가 인정됩니다. 권고사항이 기록됩니다.

**4. 결정**
여러분이 최종 결정을 내립니다 (또는 가장 강력한 증거를 바탕으로 시스템이 결정합니다). **DecisionRecord**가 결정, 근거, 참여자와 함께 `.geas/decisions/`에 기록됩니다.

## 출력

각 debate는 다음이 담긴 **DecisionRecord**를 생성합니다:

- **결정** — 어떤 옵션이 선택되었고 왜 선택되었는지
- **검토된 옵션** — 각각의 장점, 단점, 위험
- **트레이드오프** — 무엇을 얻고 무엇을 잃는지
- **참여자** — 어떤 에이전트가 어떤 입장을 주장했는지
- **증거** — 각 에이전트가 인용한 구체적인 기술적 사항

## 팁

- **명확하게 구성하세요** — 질문이 좋을수록 토론이 좋아집니다
- **컨텍스트를 제공하세요** — 기존 코드, 제약 조건, 시간 압박 모두 중요합니다
- **구조를 신뢰하세요** — Critic의 역할은 약한 논거에 도전하는 것입니다
- **집중하게 유지하세요** — 2–3가지 옵션이 이상적이며, 더 많으면 노이즈가 생깁니다
- **결과를 활용하세요** — DecisionRecord는 여러분의 선택에 대한 추적 가능한 증거입니다
