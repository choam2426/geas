# Geas

### 의사결정. 추적. 검증.

여러 AI 에이전트가 함께 일할 때 생기는 문제를 해결합니다 — 결정에는 근거가 남고, 과정은 추적되고, 결과물은 계약 기준으로 검증됩니다.

## 문제

AI 에이전트 여러 개를 동시에 돌리면 강력하지만, 한 가지 근본적인 문제가 있습니다. 에이전트가 늘어날수록 결정도 늘어나는데, 아무도 그걸 추적하지 않는다는 겁니다.

- **누가 무슨 결정을 했는지 모릅니다.** 에이전트 A가 기술 스택을 골랐고, B가 스키마를 설계하고, C가 구현했는데 — 왜 그런 선택을 했는지 기록이 없습니다.
- **결과물이 맞는지 확인할 수 없습니다.** 각 에이전트가 "완료"라고 하지만, 실제로 요구사항을 충족했는지 검증한 적이 없습니다.
- **문제가 생기면 되짚을 수가 없습니다.** 어떤 과정을 거쳐 이 결과가 나왔는지 추적할 방법이 없습니다.

> 켈트 신화에서 **기아스(geas)**는 영웅에게 걸리는 절대적인 서약입니다. 반드시 지켜야 하고, 어기면 파멸합니다.
>
> 이 프로젝트도 마찬가지입니다. 모든 에이전트는 **계약**에 묶여서 일합니다 — 지켜야 할 기준, 넘으면 안 되는 경계, 남겨야 할 증거가 정해져 있고, 예외는 없습니다.

## 해결

Geas는 세 가지를 해결합니다:

### 의사결정

결정을 암묵적으로 내리지 않습니다. 아키텍처 선택은 **투표**를 거치고, 의견이 갈리면 **토론**이 붙고, 결과는 **결정 기록**으로 남습니다. "모델이 그렇게 느꼈으니까"로 넘어가는 일은 없습니다.

### 추적

모든 과정이 기록으로 남습니다. 미션은 **seed spec**으로 고정되고, 작업은 **계약**으로 컴파일되고, 에이전트는 **증거**를 남기고, 상태 변화는 **이벤트 로그**에 쌓입니다. 뭐가 일어났고, 누가 했고, 왜 했는지 되짚을 수 있습니다.

### 검증

"완료"는 증거 게이트를 통과했다는 뜻입니다. "에이전트가 됐다고 함"이 아닙니다. 모든 작업은 **3단계 게이트**를 통과합니다:

| 계층 | 질문 | 방법 |
|------|------|------|
| **기계적** | 코드가 동작하는가? | 빌드, 린트, 테스트 |
| **의미론적** | 올바른 것을 만들었는가? | 요구사항 대조 |
| **제품** | 미션에 부합하는가? | 제품 리뷰 판정 |

## 빠른 시작

### 준비물

- [Claude Code CLI](https://claude.ai/code) 설치 및 인증
- 선택: 팀 작업 현황을 Linear에서 보려면 [Linear](https://linear.app) API 키
- 선택: Linear 연동 스크립트를 쓰려면 Python 3.6+

### 1. 플러그인 설치

```bash
/plugin marketplace add choam2426/geas
/plugin install geas@choam2426-geas
```

또는 `/plugin`을 입력해 인터랙티브 UI를 열고, **Marketplaces**에서 `choam2426/geas`를 추가한 뒤 **Discover**에서 설치합니다.

### 2. 미션을 설명합니다

```text
공유 링크로 초대할 수 있는 실시간 투표 앱을 만들어줘.
```

Compass(오케스트레이터)가 요구사항을 정리하고, 계약을 만들고, 전문 에이전트를 배치하고, 증거 게이트로 결과를 검증합니다.

### 3. 과정을 확인합니다

- **Linear 연결 시**: 각 이슈에서 팀의 댓글과 상태 전환을 확인합니다.
- **Linear 미연결 시**: `.geas/`에서 seed, 계약, 증거, 결정 기록을 확인합니다.

## 동작 방식

```text
사용자 의도 (자연어)
    → Intake Gate (질문으로 요구사항 정제 → seed.json 고정)
    → Task Compiler (seed → 요구사항이 포함된 TaskContract)
    → 에이전트 실행 (ContextPacket → 작업 → EvidenceBundle)
    → Evidence Gate (3계층: 기계적 → 의미론적 → 제품)
    → Pass: 배포  /  Fail: 수정-재검증 루프
```

모든 기록은 `.geas/`에 남습니다 — seed, 계약, 증거, 결정, 이벤트 로그. 전체 과정을 되짚을 수 있는 근거입니다.

## 팀의 작업을 보고, 개입하기

에이전트 활동은 외부 도구에 실시간으로 올라갑니다. 사람은 여기서 진행 상황을 보고, 필요하면 댓글로 끼어들 수 있습니다.

**현재**: [Linear](https://linear.app) (선택) | **미래**: 자체 대시보드

```
[Compass] 작업 시작. Pixel에게 할당.
[Palette] 모바일 퍼스트 레이아웃. 세로 카드 스택.
[사람] 파이차트 대신 막대그래프로 해줘.          ← 사람의 개입
[Forge] 동의. CSS-only 막대그래프.
[Pixel] 구현 완료. 5개 컴포넌트.
[Sentinel] QA: 5/5 기준 통과.
[Compass] Evidence Gate PASSED.
[Nova] Ship.
```

사람이 남긴 댓글은 다음 에이전트한테 자동으로 전달됩니다. 팀에 끼어드는 방법이 이겁니다.
